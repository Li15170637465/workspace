正确使用 GitHub 仓库管理代码需要遵循标准的工作流程，并处理好各种"挂起"状态（未提交/未推送的更改）。以下是完整的操作指南：

---

### ? 正确使用 GitHub 仓库的工作流程

#### 1. 日常开发循环

```mermaid
graph LR
A[修改代码] --> B[暂存更改]
B --> C[提交到本地仓库]
C --> D[推送到GitHub]
D --> E[新任务/修复]
```

#### 2. 具体操作步骤（在 VSCode 中）：

1. **修改代码后**：

   - 打开源代码管理视图 `Ctrl+Shift+G`
   - 所有更改文件会显示在"更改"区域
2. **暂存更改**：

   - 点击文件旁的 `+` 图标暂存单个文件
   - 或点击"更改"标题旁的 `・・・` > `暂存所有更改`
3. **提交更改**：

   - 在提交消息框输入有意义的描述（如"修复登录页面布局"）
   - 点击顶部 ? 图标提交（或 `Ctrl+Enter`）
   - ? 提交仅保存到**本地仓库**
4. **推送到 GitHub**：

   - 点击状态栏左下角的同步图标 ?
   - 或按 `Ctrl+Shift+P` 输入 `Git: Push`

### ?? 处理"挂起"状态的解决方案

#### 场景 1：未暂存的更改（工作区修改）

- **表现**：文件显示在"更改"区域，图标为 `U`
- **处理**：
  1. 右键文件 → 选择"放弃更改"丢弃修改
  2. 或暂存后提交

#### 场景 2：已暂存未提交（暂存区）

- **表现**：文件显示在"暂存的更改"区域，图标为 `A`(新增)/`M`(修改)
- **处理**：
  ```bash
  # 取消单个文件的暂存
  git restore --staged 文件名

  # 取消所有暂存
  git reset
  ```

#### 场景 3：已提交未推送（本地仓库）

- **表现**：状态栏显示"↑n"（n个未推送提交）
- **处理**：
  1. 点击状态栏同步图标 ?
  2. 如果推送失败：
     ```bash
     # 先拉取远程更新（避免冲突）
     git pull origin main

     # 再推送
     git push origin main
     ```

#### 场景 4：冲突文件

- **表现**：文件显示为 `C`(冲突) 或 `!`
- **处理**：
  1. 双击冲突文件
  2. 使用 VSCode 的冲突解决工具：
     - 选择"接受当前更改"（你的版本）
     - 或"接受传入更改"（远程版本）
     - 或手动编辑合并
  3. 解决后暂存并提交

### ?? 代码管理最佳实践

1. **分支策略**：

   ```bash
   # 创建新功能分支
   git checkout -b feature/新功能名称

   # 完成开发后合并到主分支
   git checkout main
   git merge feature/新功能名称
   ```
2. **.gitignore 配置**：

   - 在项目根目录创建 `.gitignore` 文件
   - 添加需要忽略的文件/文件夹：
     ```
     # 示例内容
     node_modules/
     .env
     *.log
     .DS_Store
     ```
3. **定期同步**：

   - 每天开始工作前：
     ```bash
     git pull origin main
     ```
   - 每天结束工作时：
     ```bash
     git add .
     git commit -m "每日备份"
     git push origin main
     ```
4. **提交规范**：

   - 类型前缀：
     ```
     feat:     新功能
     fix:      错误修复
     docs:     文档更新
     style:    代码格式
     refactor: 代码重构
     ```

### ? 常见问题处理

1. **推送被拒绝**：

   ```bash
   # 强制推送（仅限个人分支）
   git push -u origin main --force
   ```
2. **误提交大文件**：

   ```bash
   # 从历史中清除大文件
   git filter-branch --tree-filter 'rm -f 大文件路径' HEAD
   git push origin main --force
   ```
3. **撤销本地提交**：

   ```bash
   # 撤销最近1次提交（保留更改）
   git reset HEAD~1 --soft

   # 完全丢弃最近1次提交
   git reset HEAD~1 --hard
   ```

### ? 状态图标速查表

| 图标    | 含义          | 操作         |
| ------- | ------------- | ------------ |
| `U`   | 未跟踪文件    | 暂存或忽略   |
| `M`   | 已修改文件    | 暂存后提交   |
| `A`   | 新添加文件    | 暂存后提交   |
| `D`   | 已删除文件    | 提交删除操作 |
| `C`   | 冲突文件      | 手动解决冲突 |
| `↑n` | n个提交未推送 | 点击同步图标 |
| `↓n` | n个更新未拉取 | 先拉取再推送 |

在 VSCode 中，使用源代码管理视图（`Ctrl+Shift+G`）可以直观地管理所有状态。养成"小步提交"的习惯（每完成一个小功能就提交一次），能有效减少挂起状态的困扰。
